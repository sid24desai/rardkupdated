Terminology

Reply Definition
A reply definition is the entire thing you're working on. This includes triggers, replies, user-specific overrides, and any additional settings.
Trigger
A trigger is what the user will type that the bot will recognize to activate the reply definition. A trigger is the only required element of a reply definition. You can specify multiple triggers.
Trigger Requires Bot Name
This is used if you'd like to trigger the bot without including the bot name keyword (see below) in the trigger(s) for that reply definition, but still only want to trigger this reply definition if the bot's name is included in the message.
Specific Channels
This allows you to specify Channel IDs which will mean that the reply definition will only trigger in those specific channels. You can specify as many channels as you'd like. You must use the Channel ID .
Specific Users
This allows you to specify User IDs which will mean that the reply definition will only trigger for those specific users. You can specify as many users as you'd like. You must use their User ID .
Reply
A reply is what the bot will respond when it recognizes the trigger. You can specify multiple replies, they will be chosen randomly.
Mention Author When Replying
The Mention Author setting is used to determine whether or not the user who triggered the bot's reply will be mentioned in the reply.
Reactions
You can add emoji reactions to your reply definitions. You can only include one (1) emoji at a time (add multiple reactions to have multiple emoji reactions), and custom emotes are not supported at this time.
Priority
The bot will look at your reply definitions and choose one from top to bottom. That is, if you have two reply definitions with the same trigger(s), it will choose whichever is first in the reply definitions list. You can use the up and down arrows on a reply definition to adjust their priority.
